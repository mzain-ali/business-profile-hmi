# HMI Parts — Business Profile (Next.js + React)

The HMI Parts business profile, built as a real Next.js/React app instead of
a design-tool file. Same 10-page landscape document as before — cover, about,
why-us, services, products, stock coverage, brands, locations, FAQ, contact —
but now every page is a component, all content lives in one file, and the
final PDF is generated by exporting the live page with Playwright (real
Chromium) instead of a static design export.

## 1. Run it

```bash
npm install
npm run dev
```

Open **http://localhost:3000/print** — that route is the actual document,
laid out at its real fixed size (13.333in × 7.5in per page, matching the
landscape format of the original profile). The homepage at `/` just links to it.

## 2. Edit content (text, numbers, phone numbers, FAQ, brands, stats)

Everything lives in **`src/lib/content.ts`** — one file, typed objects, no
JSX editing required. Look for `// ←` comments marking things that still need
real data:

- `coverage.bars` — stock-line figures are illustrative placeholders
- `locations.branches[1].photo` — Barka currently uses an interior photo,
  not a storefront shot (the only "Barka" photo available showed a different
  company's signage and was deliberately excluded — see the note in the file)

## 3. Change brand colors / fonts

- Colors, spacing: CSS variables at the top of **`src/app/globals.css`**
  (`--orange`, `--navy`, `--ink`, etc.) — every component reads only these
  variables, so changing them restyles the whole document.
- Fonts: `src/app/layout.tsx` imports `Sora` (display) and `Inter` (body) via
  `next/font/google`.
- Shapes: `src/components/profile/shapes.module.css` holds the chamfered
  corners, icon boxes, and badge styles shared by every page.

## 4. Add or reorder a page

1. Copy an existing page pair, e.g. `src/components/profile/ServicesPage.tsx`
   + `.module.css`.
2. Add its content block to `src/lib/content.ts`.
3. Add one import + one line in `src/app/print/page.tsx`, and update the
   `pageNumber` props on the pages around it.

## 5. Generate the PDF

The PDF is produced by pointing a real headless browser (Playwright) at the
`/print` route and printing it — this preserves fonts, the chamfered
clip-path shapes, and photo placement exactly as rendered, unlike a
design-tool export.

```bash
npx playwright install chromium   # first time only
npm run build && npm run start    # in one terminal — leaves a server running
npm run pdf                       # in another terminal — writes the PDF
```

This produces `HMI-Parts-Business-Profile.pdf` in the project root.

> **Note on this environment:** the sandbox this project was built in
> couldn't reach Playwright's browser-download CDN, so the PDF export
> itself was not re-verified by running `npm run pdf` here — only the
> Next.js build/render was verified. Run the steps above in your own
> environment (any normal machine with internet access) to generate the PDF;
> the underlying HTML/CSS is identical to the version already verified
> working via WeasyPrint.

## 7. Tech stack

- Next.js 14 (App Router) + TypeScript
- CSS Modules consuming a shared token system (`globals.css` + `shapes.module.css`)
- No animation/JS framework needed — this is a static print document, not an
  interactive site, so there's no GSAP or client-side state here
- Playwright for the PDF export (real Chromium — full CSS support, unlike
  lighter-weight HTML-to-PDF converters)

## 8. Project structure

```
src/
├── app/
│   ├── layout.tsx       # fonts, metadata
│   ├── page.tsx         # simple index, links to /print
│   ├── globals.css      # design tokens (--orange, --navy, --ink...)
│   ├── print.css        # @page size rule for the PDF export
│   └── print/
│       └── page.tsx     # composes all 10 pages in order — THE deliverable
├── components/
│   └── profile/
│       ├── PageShell.tsx        # fixed-size (13.333in × 7.5in) page wrapper
│       ├── Icons.tsx            # shared SVG icon sprite + <Icon>
│       ├── shapes.module.css    # chamfer corners, icon boxes, badges
│       └── CoverPage.tsx, AboutPage.tsx, ...  # one component per page
└── lib/
    └── content.ts        # ALL copy, numbers, links — the single edit point
scripts/
└── generate-pdf.mjs       # Playwright export script
public/assets/              # logo + real branch photography
```
